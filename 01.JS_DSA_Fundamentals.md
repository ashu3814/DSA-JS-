## 1. Variables

Variables are containers for storing data values. In JavaScript, you can declare variables using `var`, `let`, or `const`.

### Keywords for Declaration

| Keyword | Scope          | Hoisting      | Re-declaration | Re-assignment | Preferred Use         |
| :------ | :------------- | :------------ | :------------- | :------------ | :-------------------- |
| `var`   | Function/Global| Yes (undefined)| Yes            | Yes           | Avoid (legacy)      |
| `let`   | Block `{}`     | No (TDZ*)     | No (in scope)  | Yes           | For re-assignable vars|
| `const` | Block `{}`     | No (TDZ*)     | No (in scope)  | No**          | For constant values   |

\* TDZ: Temporal Dead Zone - you cannot access the variable before its declaration.
\*\* For `const` with objects/arrays, the reference is constant, but the object/array's content can be modified.

### Naming Conventions
*   Start with a letter, underscore `_`, or dollar sign `$`.
*   Subsequent characters can also be digits (0-9).
*   Case-sensitive (`myVar` and `myvar` are different).
*   Use **camelCase** (e.g., `firstName`, `totalAmount`).
*   Choose descriptive names.

### Examples

```javascript
// Using var (avoid in modern JS)
var oldSchoolVar = "I am old school";
console.log(oldSchoolVar); // Output: I am old school

// Using let (for variables that might be reassigned)
let age = 30;
console.log(age); // Output: 30
age = 31;
console.log(age); // Output: 31

// Using const (for variables that should not be reassigned)
const birthYear = 1993;
console.log(birthYear); // Output: 1993
// birthYear = 1994; // This would cause an error: Assignment to constant variable.

// const with objects/arrays
const person = { name: "Alice" };
person.name = "Bob"; // This is allowed, modifying the object's property
console.log(person.name); // Output: Bob
// person = { name: "Charlie" }; // This would cause an error, reassigning the const reference

const numbers = [1, 2, 3];
numbers.push(4); // This is allowed, modifying the array's content
console.log(numbers); // Output: [1, 2, 3, 4]
// numbers = [5, 6]; // This would cause an error

2. Data Types

JavaScript is a dynamically typed language. This means you don't have to declare the type of a variable. The type is determined automatically at runtime.

Primitive Data Types

These are immutable (their values cannot be changed once created).

Type	Description	Example	typeof returns
String	Sequence of characters (text)	"Hello", 'World'	"string"
Number	Numeric values (integers, floats, NaN, Infinity)	10, 3.14, NaN	"number"
Boolean	Logical entity, can be true or false	true, false	"boolean"
Undefined	Variable declared but not assigned a value	let x; (x is undefined)	"undefined"
Null	Intentional absence of any object value	let y = null;	"object" (quirk)
Symbol	(ES6) Unique and immutable primitive value	Symbol('id')	"symbol"
BigInt	(ES2020) For integers larger than 2^53 - 1	12345678901234567890n	"bigint"
Non-Primitive (Reference) Data Type

These are mutable (their values can be changed).

Type	Description	Example	typeof returns
Object	Collection of key-value pairs (properties)	{ name: "Alice", age: 30 }	"object"
Array	Ordered list of values	[1, "apple", true]	"object"
Function	A block of code designed to perform a task	function greet() { console.log("Hi"); }	"function"
Checking Data Types

The typeof operator is used to find the data type of a JavaScript variable.

let message = "Hello";
let count = 100;
let isActive = true;
let data = null;
let notAssigned;
let person = { name: "John", city: "New York" };
let scores = [90, 85, 92];

console.log(typeof message);     // "string"
console.log(typeof count);       // "number"
console.log(typeof isActive);    // "boolean"
console.log(typeof data);        // "object" (historical quirk for null)
console.log(typeof notAssigned); // "undefined"
console.log(typeof person);      // "object"
console.log(typeof scores);      // "object" (arrays are objects)
console.log(typeof function(){});// "function"

switch Statement

Used to perform different actions based on different conditions for a single variable.
It's often an alternative to a long if...else if...else chain.

let day = "Monday";
let activity;

switch (day) {
  case "Monday":
    activity = "Work";
    break; // Important! Exits the switch block.
  case "Tuesday":
    activity = "Study DSA";
    break;
  case "Saturday":
  case "Sunday": // Multiple cases can share the same code block
    activity = "Relax";
    break;
  default: // Optional: executed if no case matches
    activity = "Do something else";
}
console.log(`On ${day}, I will ${activity}.`); // On Monday, I will Work.

Key switch points:

The switch expression is evaluated once.

The value of the expression is compared with the values of each case.

If there is a match, the associated block of code is executed.

The break statement stops the execution inside the switch block. If omitted, execution "falls through" to the next case.

The default keyword specifies the code to run if there is no case match.

5. Loops

Loops are used to execute a block of code repeatedly as long as a certain condition is true, or for a specific number of times.

Loop Comparison
Loop Type	When to Use	Syntax Overview
for	When you know the number of iterations beforehand.	for (initialization; condition; increment/decrement)
while	When the number of iterations is not known, and depends on a condition.	while (condition)
do...while	Similar to while, but guarantees the code block executes at least once.	do { ... } while (condition);
for...in	For iterating over the enumerable string properties (keys) of an object.	for (key in object)
for...of	(ES6) For iterating over iterable objects (values of Arrays, Strings, Maps, Sets).	for (value of iterable)
for Loop
// Print numbers from 0 to 4
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Output:
// 0
// 1
// 2
// 3
// 4

// Iterate over an array
let fruits = ["Apple", "Banana", "Cherry"];
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}
// Output:
// Apple
// Banana
// Cherry

Loop Control: break and continue

break: Terminates the current loop (or switch statement) and transfers program control to the statement following the terminated statement.

continue: Skips the current iteration of a loop and continues with the next iteration.

// break example
for (let i = 0; i < 10; i++) {
  if (i === 5) {
    break; // Stop the loop when i is 5
  }
  console.log(i); // 0, 1, 2, 3, 4
}

// continue example
for (let i = 0; i < 5; i++) {
  if (i === 2) {
    continue; // Skip iteration when i is 2
  }
  console.log(i); // 0, 1, 3, 4
}
